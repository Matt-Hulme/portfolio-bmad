## Dev Notes

### Architecture Context

**Design System** [Source: prd/user-interface-design-goals.md]

**Design Direction:** "Hacker Minimalist"
- **Color Palette:** Muted green (#7fda89) as primary accent on very dark background (#0f0f0f)
- **Typography:** Monospace fonts (SF Mono, Courier New) for terminal feel
- **Visual Elements:** Left border accents on cards, `$` prompt on tech lines, subtle hover effects
- **Visual Hierarchy:** Green used strategically for interactive elements and accents—most content stays in comfortable gray tones
- **Philosophy:** Terminal touches without overwhelming; content over decoration

**Tailwind Configuration** [Source: architecture.md#9.6]
- Use Tailwind 4.x configuration format
- Path alias already configured: `@` → `./src`
- Breakpoints: mobile-first (sm: 640px, md: 768px, lg: 1024px, xl: 1280px, 2xl: 1536px)

**shadcn/ui Integration** [Source: Story 1.1]
- Already initialized in Story 1.1
- Components path: `src/components/ui/`
- Button component already installed
- Dialog component available (already installed in 1.1)
- Use `cn()` helper for conditional classes

### Color Palette Details

**Primary Colors:**
```typescript
{
  primary: '#7fda89',      // Muted green - interactive elements
  background: '#0f0f0f',   // Very dark background
}
```

**Gray Scale (Suggested):**
```typescript
{
  gray: {
    50: '#fafafa',
    100: '#f5f5f5',
    200: '#e5e5e5',
    300: '#d4d4d4',
    400: '#a3a3a3',
    500: '#737373',
    600: '#525252',
    700: '#404040',
    800: '#262626',
    900: '#171717',
  }
}
```

### Typography Tokens

**Font Stack:**
```css
font-family: 'SF Mono', 'Monaco', 'Inconsolata', 'Courier New', monospace;
```

**Type Scale (Suggested):**
- h1: 3rem (48px)
- h2: 2.25rem (36px)
- h3: 1.875rem (30px)
- h4: 1.5rem (24px)
- h5: 1.25rem (20px)
- h6: 1rem (16px)
- body: 1rem (16px)
- small: 0.875rem (14px)

### Coding Standards [Source: architecture.md#16.1]

**React:**
- Functional components only
- Named exports (not default)
- TypeScript interfaces for all props
- Event handlers prefixed with `handle`

**Styling:**
- TailwindCSS utility classes
- Consistent class ordering (via prettier-plugin-tailwindcss)
- Use `cn()` helper for conditional classes (from lib/utils.ts)

**Import Order:**
1. React imports
2. Third-party libraries
3. Internal modules (@ alias)
4. Relative imports
5. Styles

### Testing Requirements

**Unit Tests:**
- Container component renders with correct classes
- Typography styles apply correctly
- Color tokens are accessible

**Visual Testing:**
- Component showcase page displays all components
- Responsive breakpoints work correctly
- Dark theme applies consistently
